<task type="auto">
  <name>Task 3: Fix stale loader cache on profile edit page with staleTime: 0</name>
  <files>src/routes/_protected/profile/edit.tsx</files>
  <action>
Root cause (from UAT diagnosis): TanStack Router reuses the loader cache on SPA navigation to `/profile/edit`, so `getOwnProfile()` is not re-called. The form initialises with `defaultValues` from the stale cached data, causing `fieldVisibility` to revert to the pre-save values. A browser reload forces a fresh fetch, which is why visibility resets are only visible before reload.

Fix: add `staleTime: 0` to the route definition so TanStack Router always re-runs the loader when navigating to this route.

Current route definition in `src/routes/_protected/profile/edit.tsx` (lines 11-14):
```tsx
export const Route = createFileRoute('/_protected/profile/edit')({
  loader: () => getOwnProfile(),
  component: EditPage,
})
```

Change to:
```tsx
export const Route = createFileRoute('/_protected/profile/edit')({
  loader: () => getOwnProfile(),
  staleTime: 0,
  component: EditPage,
})
```

No other changes to this file.
