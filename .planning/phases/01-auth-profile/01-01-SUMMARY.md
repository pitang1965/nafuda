## Issues Encountered
- **Auth gate: Neon DATABASE_URL not configured** — `pnpm db:migrate` was attempted and failed (placeholder URL in .env.local). User must create Neon project in ap-northeast-1 (Tokyo) and set DATABASE_URL before running `pnpm db:migrate`. See user_setup section in 01-01-PLAN.md for exact steps.

## User Setup Required

Before running `pnpm db:migrate`, complete these steps:

1. **Create Neon project:**
   - Go to https://console.neon.tech → New Project
   - Select region: `ap-northeast-1` (Tokyo)
   - Copy the connection string (pooled, with `?sslmode=require`)

2. **Update .env.local:**
   ```
   DATABASE_URL=postgresql://<user>:<pass>@<host>.neon.tech/<db>?sslmode=require
   BETTER_AUTH_SECRET=<run: openssl rand -base64 32>
   ```

3. **Run migrations:**
   ```bash
   pnpm db:migrate
   ```
   Expected: Tables `url_ids`, `personas`, `sns_links` created in Neon

4. **Generate Better Auth schema (Plan 01-02):**
   ```bash
   pnpm auth:generate
   pnpm db:migrate
   ```
   Expected: Tables `user`, `session`, `account`, `verification` created in Neon
