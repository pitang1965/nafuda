**events テーブル:**
```typescript
export const events = pgTable('events', {
  id: uuid('id').primaryKey().defaultRandom(),
  slug: text('slug').notNull().unique(),         // URL-safe identifier 例: "animejapan-20260405"
  name: text('name').notNull(),                  // イベント名（表示用）
  venueName: text('venue_name').notNull(),        // 会場名
  eventDate: timestamp('event_date', { withTimezone: true }).notNull(),
  createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
})
```

**event_checkins テーブル:**
```typescript
export const eventCheckins = pgTable('event_checkins', {
  id: uuid('id').primaryKey().defaultRandom(),
  eventId: uuid('event_id').notNull().references(() => events.id),
  personaId: uuid('persona_id').notNull().references(() => personas.id),
  userId: text('user_id').notNull(),             // Better Auth user.id（認証チェック用）
  // GPS: point mode 'xy' → { x: longitude, y: latitude }。ユーザーが拒否した場合 null
  gpsCoordinates: point('gps_coordinates', { mode: 'xy' }),
  checkedInAt: timestamp('checked_in_at', { withTimezone: true }).defaultNow().notNull(),
  checkedOutAt: timestamp('checked_out_at', { withTimezone: true }),  // NULL = チェックイン中
  createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
})
```

追加後、マイグレーション生成と適用:
```bash
pnpm db:generate
pnpm db:migrate
```

**4関数の実装:**

**1. checkinToEvent** (POST, 認証必須):
- inputValidator: `{ slug, eventName, venueName, eventDate, personaId, gpsCoordinates? }`
  - slug: `z.string().min(1).max(100).regex(/^[a-zA-Z0-9-]+$/)`（URL-safe）
  - eventName/venueName: `z.string().min(1).max(100)`
  - eventDate: `z.string().datetime()`
  - personaId: `z.string().uuid()`
  - gpsCoordinates: `z.object({ x: z.number(), y: z.number() }).optional()`
- 処理:
  1. `getRequest()` + `auth.api.getSession` で認証チェック（未認証は `throw new Error('Unauthorized')`）
  2. personaId の owner が session.user.id か確認（他人のペルソナへのチェックイン防止）
  3. 既存アクティブチェックイン（checkedOutAt IS NULL）があれば auto-checkout（checkedOutAt = new Date()）
  4. slug で events を SELECT — 存在しなければ INSERT（slug 衝突時は既存を使用）
  5. event_checkins に INSERT して新レコードを返す

**2. checkoutFromEvent** (POST, 認証必須):
- inputValidator: `{ checkinId: z.string().uuid() }`
- 処理:
  1. 認証チェック
  2. checkinId で SELECT して userId が session.user.id か確認
  3. `checkedOutAt = new Date()` で UPDATE

**3. getActiveCheckin** (GET, 認証必須):
- inputValidator: `{ personaId: z.string().uuid() }`
- 処理:
  1. 認証チェック
  2. `WHERE personaId = ? AND checkedOutAt IS NULL LIMIT 1`
  3. JOIN events テーブルでイベント名・会場名も返す（UI表示用）
  4. 存在しなければ null を返す

**4. getEventParticipants** (GET, 認証不要 — 公開エンドポイント):
- inputValidator: `{ slug: z.string() }`
- 処理:
  1. slug で events を SELECT — 存在しなければ null を返す
  2. JOIN: `event_checkins INNER JOIN personas ON personaId INNER JOIN urlIds ON personas.userId = urlIds.userId`
  3. WHERE: `eventId = ? AND checkedOutAt IS NULL AND dojinReject = false AND isPublic = true`
  4. SELECT のみ: `checkinId, personaId, displayName, avatarUrl, urlId, shareToken`（SNSリンク・fieldVisibility は絶対含めない）
  5. `{ event: eventRow, participants }` を返す

**CRITICAL ルール（RESEARCH.md Pitfall より）:**
- `getRequest()` を必ず使用（`context.request` は TanStack Start v1.167 に存在しない）
- `dojinReject` フィルタは SQL WHERE 句で行う（クライアントサイド filter() 禁止）
- `checkedOutAt IS NULL` フィルタも SQL WHERE 句で行う
- `getEventParticipants` は SNS リンクや fieldVisibility を絶対に返さない
