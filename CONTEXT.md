# なふだ — ドメイン用語集

このファイルはドメイン語彙の唯一の定義源。実装の詳細や仕様は書かない。

---

## 用語定義

### なふだ
ユーザーが他者に見せるプロフィールカード。アプリ名でもある。コード上は **Persona** と呼ぶ。UI上では常に「なふだ」と表記する（「ペルソナ」はユーザーに見せない）。1ユーザーが複数持てる（例：「婚活用なふだ」「車趣味用なふだ」）。「マイなふだ」は自分のなふだ一覧を指す。

### ペルソナ (Persona)
「なふだ」のコード上の名称。`personas` テーブルに格納。1ユーザーにつき1つが `isDefault = true` のデフォルトペルソナとなり、チェックインの主体になる。コネクションは「つながる」時にユーザーが選んだペルソナが主体になる（複数ペルソナがある場合はピッカーを表示、1つのみの場合は自動選択）。UI上では「なふだ」と呼ぶ。

### ラベル (Label)
なふだの用途を自分だけが識別するための名前。**オーナーにのみ表示され、公開プロフィールには出ない**。「推し活用」「婚活用」「仕事用」など自由記述。コード上は `label`。`personas.label` に格納。任意項目（未設定の場合は displayName のみ表示）。ウィザードで作成する初回なふだにはデフォルト値「メイン」が入力欄にセットされる。なふだ作成時に入力欄を用意し、未設定のなふだには設定を促すリンクを表示する。

### UrlId
ユーザーに1つ割り当てられる英数字識別子。**システムが生成するランダム文字列**であり、ユーザーは選択できない。変更不可。`url_ids` テーブルに格納。`/u/{urlId}` 単独のルートは存在しない。すべてのなふだへのアクセスは ShareToken を伴う URL 経由のみ。UrlId 自体はアクセス制御に関与しない（推測耐性は ShareToken が単独で担う）。

### ShareToken
ペルソナごとに発行されるランダムな16進数トークン。QRコードのURLに埋め込まれ、プロフィール参照に使われる。デフォルトペルソナも含め、すべてのペルソナが固有の ShareToken を持つ。`personas.share_token` に格納。

### プロフィールQR
`/u/{urlId}/p/{shareToken}` 形式の永続URLをエンコードした画像。「なふだを見せる」で使用する。インターネット公開・印刷も可能。スキャンしてもコネクションは生成されない。

### つながりQR
「なふだを交換する」で生成される短命（15分有効）のQRコード画像。スキャンした時点でサーバーに保留中の招待が記録される。コネクションはスキャン者が「つながる」をタップした時点で双方向・即時に成立する。

### イベント (Event)
参加者がチェックインできる時間と場所のある記録単位。2種類ある：

**企画イベント** — 誰かが事前に作成した催し（ライブ・コンサート・ファンミ・オフ会・勉強会等）。`/events/new` から明示的に作成する。会場名（`venueName`）必須。公開イベントページ（`/e/{shareToken}`）を持つ。

**即時イベント** — 「なふだを交換する」フロー中にその場で作成する出会いの文脈記録（「バーナイト＠立川」等）。`events.isInstant = true`。イベント名と GPS 座標のみで作成可能（会場名は任意）。スラッグは自動生成。作成と同時に作成者がチェックインした状態になる。コネクションが成立しなくても作成される。**公開ページを持たない**（他者からアクセス不可）。作成者のイベント一覧には企画イベントとは別セクションで表示される。コネクション相手のイベント一覧には表示されない（コネクションの文脈テキストとしてのみ参照可能）。

両種別とも `events` テーブルに格納。作成時にどのペルソナで作ったかを `hostPersonaId` として記録する。`events.gpsCoordinates`（任意）に GPS 座標（緯度・経度）を保存する。

**イベントに含まれないもの：**
- 個人の聖地巡礼・撮影地訪問（個人の行動であり、主催者のいる催しではない）
- 趣向・コミュニティ（「○○ファン」「車中泊グループ」等）

聖地を使った企画催し（「○月○日に聖地○○に集まれ！」等）は主催者が作成したイベントとして扱う。個人の聖地訪問を記録したい場合は推しタグで代替する。

### チェックイン (Checkin)
ペルソナがイベントに「入る」アクション。`checkedOutAt IS NULL` の状態がアクティブ。新しいチェックイン時に直前のチェックアウトが自動記録される（同時に複数イベントへの参加不可）。複数のなふだを持つユーザーにはペルソナピッカーを表示し、どのなふだとして参加するかを選ぶ（1つのみの場合は自動選択）。同一ユーザーが同一イベントに別のなふだで再チェックインした場合、以前のなふだのチェックイン記録は削除される（1ユーザー = 1エントリ）。イベント参加者一覧は開催前・開催後を問わず参加を表明したすべてのペルソナを表示する。`event_checkins` テーブルに格納。

### コネクション (Connection)
2つのペルソナ間の出会いの記録。生成経路は**なふだを交換する**（つながりQR経由）または**つながりリクエスト**（イベント参加者一覧経由の承認）の2種類のみ。いずれの場合もA→BとB→Aの2行が同時に生成される（即時双方向）。削除は非対称で、各自が自分の行を独立して削除できる（相手の行には影響しない）。複数のなふだを持つユーザーには接続時にペルソナピッカーを表示する（1つのみの場合は自動選択）。各コネクションには「出会いの文脈」（いつ・どこで）が付与される。文脈の付与経路は以下の通り：

- **①自動（イベントチェックイン）** — QR を**発行した側**（A）が**企画イベント**にチェックイン中の場合、そのイベント情報が双方のコネクション行に自動付与される。
- **②自動（即時イベント作成）** — 「なふだを交換する」タップ後、QR 表示前に「今どこで？」スクリーン（スキップ可）が表示され、イベント名を入力すると即時イベントが作成されてAがチェックインする。GPS 座標は A の端末から自動取得（位置情報許可が必要）し `events.gpsCoordinates` に保存。このチェックインが①の文脈として機能するが、**即時イベントの文脈（イベント名）は A のコネクション行にのみ付与される**（B のコネクション行には付与しない）。B はスキャン時にそのメモを知らないため、B は後付け編集（③）で自分の言葉で文脈を記録できる。スキップした場合は文脈なしで QR を表示する。
- **③後付け編集** — 文脈が空のコネクション（イベント参照なし）に限り、コネクション成立後に場所名・イベント名をフリーテキストで書き足し・再編集できる。イベントチェックイン由来で自動付与された文脈は編集不可（事実の記録として固定）。
- **④プライベートメモ** — 自分だけが見られる相手についての覚え書き（`connections.privateMemo`）。コネクション成立後に追記・編集できる。

なふだ交換時の既存イベント選択は設計上行わない（誤選択リスクのため）。ただし A がすでにイベントにチェックイン中であれば、そのアクティブチェックインが自動的に文脈として使われる。コネクション一覧の「どのなふだとして」表示には、接続時点のラベルではなく**現在のラベル**を使う（ペルソナテーブルをJOINして取得）。ラベルを変更すると過去のコネクション表示も更新される。`connections` テーブルに格納。

### なふだを見せる
自分のプロフィールQRを他者に提示するアクション。永続URLのためインターネット公開・印刷もできる。閲覧が目的であり、コネクションは生成されない。

### なふだを交換する
つながりQRを生成・提示し、相手がスキャンして「つながる」をタップするアクション。紙の名刺交換に相当する。コネクションが双方向・即時に成立する。

**フロー：**
1. 「なふだを交換する」をタップ
2. 「今どこで？」スクリーンが表示される（スキップ可）。イベント名を入力すると即時イベントが作成されてチェックイン状態になる。GPS 座標は自動取得。スキップすると文脈なし。
3. QR（つながりQR）が表示される。
4. 相手がスキャンして「つながる」を押すとコネクション成立。A 側には「つながりました！」通知。
5. 「**交換しない**」を選ぶとつながりQRトークンが即時削除される。「**交換予定**」または通知後の「**閉じる**」を選ぶとトークンはそのまま（15分有効）でシートを閉じる。

### つながりリクエスト (Connection Request)
イベント参加者一覧からAがBに送るコネクション申請。Bが承認するとコネクションが双方向・即時に成立する。イベント終了から14日で自動失効。拒否ボタンは存在しない（承認か無視のみ）。失効しても双方に通知しない。

### 保留中の招待 (Pending Invite)
つながりQRをスキャンした時点でアカウントを持たないユーザーに対してサーバーが作成する一時的なDBレコード。つながりQR URL（15分有効）とは独立して存在するため、URLが期限切れになっても有効。アカウントあり（ペルソナなし）の場合はウィザードを同一セッション内で完結できるためこのレコードは不要。アカウントなしの場合のみ作成し、有効期限はスキャンから48時間（アカウントを作る猶予期間）。ウィザードは1つのなふだしか作れないため、招待適用時のペルソナは常にデフォルトペルソナ（自動選択）。アカウントおよびペルソナ作成後に自動適用され、コネクションが双方向・即時に成立する。

### 同担拒否 (Dojin Reject)
同じ推しを応援するファン（同担）との交流を望まないというユーザーの意思表示。`personas.dojin_reject = true` の場合、そのペルソナはイベント参加者一覧から非表示になる。

### 推しタグ (Oshi Tags)
ペルソナに紐づく自由記述のタグ配列。応援しているアーティスト・キャラクター・作品などを登録する。任意項目（未設定のまま使用できる）。`personas.oshi_tags` に格納。推し活は[[打ち出し]]上「最も濃い実例」として扱われるユースケースだが、表の顔ではない（→ ADR-0009）。

### 打ち出し (Positioning)
アプリがユーザーに見せる「表の顔」＝ヒーロー・第一印象・SEOの主軸。**「その場で会った“仕事じゃない”人と、SNSでつながる」**を核に据える。対象は名刺を持たない／出さない場での出会い（勉強会・趣味のオフ会・懇親会やパーティー・バーや居酒屋での出会い等）。**ビジネス交流会は対象外**（紙名刺＋Eight の縄張りであり戦場にしない）。差別化は「Eight＝仕事の名刺／なふだ＝“仕事じゃない”出会いの名刺」。推し活・趣味・コミュニティ等の汎用性は**プロダクトの射程と二次メッセージに残す**が、SEOの核には据えない。推し活は最も拡散する実例コンテンツとして一等地に配置する。この打ち出しへの転換の経緯は ADR-0009 を参照。

### 退会 (Account Deletion)
ユーザーがアカウントとすべての関連データを即時・完全に削除するアクション。UI上では「退会する」と表記。削除対象はユーザーレコード・すべてのペルソナ・SNSリンク・チェックイン履歴・コネクション（相手側の記録も含む）・UrlId。退会後に同じメールアドレスで再登録は可能だが、新しい UrlId が発行されるため以前の QR コードは無効になる。退会してもそのユーザーが作成したイベントは削除されない（他の参加者の文脈として機能するため）。

### SNSリンク (SNS Link)
ペルソナに紐づく外部サービスへのリンク。プラットフォーム種別・URL・表示順を持つ。`sns_links` テーブルに格納。対応プラットフォーム: x, instagram, tiktok, youtube, discord, line_openchat, github, spotify, facebook, minkara, linkedin, note, pixiv, other。

### 公開範囲 (Field Visibility)
ペルソナのフィールドごとの公開/非公開設定。`personas.field_visibility` に JSON で格納。デフォルトは全フィールド公開（QRを渡す行為自体が公開の意思表示という前提）。

### なふだスタイル (NafudaStyle)
なふだ（ペルソナ）の見た目を決める装飾パッケージ。背景（CSSグラデーション／単色）・フォント（Google Fonts）・縁取り（SVGデコラティブフレーム、なしもあり）の組み合わせをひとつのパッケージとして提供する。ユーザーはパッケージをまるごと選択する（個別要素のカスタマイズは不可）。コード上は文字列IDで識別し（例: `'sakura'`）、`personas.style_id` に格納する。IDの実体（CSS定義・SVGアセット）はコード側で管理する。将来的に有料パッケージを追加できるよう、スタイル定義に `isFree` フラグを持つ。現時点では全スタイルが無料（`isFree: true`）。課金ゲートを実装する際にこのフラグを使って選択制限を追加する。一部のスタイルはホログラフィック効果（`holographic: true`）を持ち、端末の傾き（DeviceOrientation API）に連動してカード上で虹色の光沢が動く。UI上では「なふだスタイル」と表記する。適用範囲は公開プロフィールページ（`/u/{urlId}/p/{token}`）。

## 重要な区別

| 混同しやすいペア | 区別 |
|---|---|
| なふだ / ペルソナ | 同じ概念。UIでは「なふだ」、コードでは「Persona」と呼ぶ。ユーザーに「ペルソナ」という語は見せない |
| UrlId / ShareToken | UrlIdはユーザー単位・変更不可の識別子で、URL構造上の名前空間（`/u/{urlId}/...`）を与えるのみ。アクセス制御は ShareToken（ペルソナ単位・QR用トークン）が単独で担う |
| プロフィールQR / つながりQR | プロフィールQRは永続・閲覧専用。つながりQRは15分有効・コネクション生成可能。同じなふだに対して別々に存在する |
| チェックイン / コネクション | チェックインは「現場にいる」状態。コネクションは「つながった」という記録 |
| なふだを見せる / なふだを交換する | 「見せる」は閲覧のみ・コネクション不発生。「交換する」は短命QRを使ったその場の名刺交換でコネクションが成立する |
| displayName / ラベル | displayNameは他者に見せる公開名。ラベルはオーナーだけが見る自己識別用の名前（最大20文字）。コネクション一覧など自分向けの画面ではラベルを優先表示し、未設定の場合は displayName にフォールバック |
| なふだスタイル / テーマ | なふだスタイルはこのアプリの用語。「テーマ」「スキン」は使わない |
| 企画イベント / 即時イベント | 企画イベントは事前作成・公開ページあり・参加者が能動的にチェックイン。即時イベントは「なふだを交換する」フロー中に作成・公開ページなし・作成者のみ一覧に表示 |
