# なふだ — ドメイン用語集

このファイルはドメイン語彙の唯一の定義源。実装の詳細や仕様は書かない。

---

## 用語定義

### なふだ
ユーザーが他者に見せるプロフィールカード。「マイなふだ」は自分のなふだを指す。アプリ名でもある。コード上は **Persona** と呼ぶ。

### ペルソナ (Persona)
「なふだ」のコード上の名称。1ユーザーが複数持てる。`personas` テーブルに格納。1ユーザーにつき1つが `isDefault = true` のデフォルトペルソナとなり、コネクション・チェックインの主体になる。

### UrlId
ユーザーに1つ割り当てられる公開英数字識別子（例: `abc123`）。プロフィール閲覧URLに使われる。変更不可。`url_ids` テーブルに格納。

### ShareToken
ペルソナごとに発行されるランダムな16進数トークン。QRコードのURLに埋め込まれ、プロフィール参照に使われる。`personas.share_token` に格納。

### QRコード
`/u/{urlId}/p/{shareToken}` 形式のURLをエンコードした画像。これを他者が読み取ることでプロフィールを閲覧できる。スキャン自体ではコネクションは記録されない。

### イベント (Event)
ライブ・聖地巡礼・オフ会などの現場。スラッグ（URL-safe文字列）で識別される。最初のチェックイン時に自動作成される（管理画面なし）。`events` テーブルに格納。

### チェックイン (Checkin)
ペルソナがイベントに「入る」アクション。`checkedOutAt IS NULL` の状態がアクティブ。新しいチェックイン時に直前のチェックアウトが自動記録される（同時に複数イベントへの参加不可）。`event_checkins` テーブルに格納。

### コネクション (Connection)
「つながる」ボタンを押した側（fromPersona）から、QRを見せた側（toPersona）への一方向の記録。双方向のつながりは2行で表現される。チェックイン中に「つながる」を実行するとイベントのコンテキスト（名前・会場・日付）が付与される。`connections` テーブルに格納。

### 同担拒否 (Dojin Reject)
同じ推しを応援するファン（同担）との交流を望まないというユーザーの意思表示。`personas.dojin_reject = true` の場合、そのペルソナはイベント参加者一覧から非表示になる。

### 推しタグ (Oshi Tags)
ペルソナに紐づく自由記述のタグ配列。応援しているアーティスト・キャラクター・作品などを登録する。`personas.oshi_tags` に格納。

### SNSリンク (SNS Link)
ペルソナに紐づく外部サービスへのリンク。プラットフォーム種別・URL・表示順を持つ。`sns_links` テーブルに格納。

### 公開範囲 (Field Visibility)
ペルソナのフィールドごとの公開/非公開設定。`personas.field_visibility` に JSON で格納。デフォルトは全フィールド公開（QRを渡す行為自体が公開の意思表示という前提）。

## 重要な区別

| 混同しやすいペア | 区別 |
|---|---|
| なふだ / ペルソナ | 同じ概念。UIでは「なふだ」、コードでは「ペルソナ」と呼ぶ |
| UrlId / ShareToken | UrlIdはユーザー単位・変更不可の公開識別子。ShareTokenはペルソナ単位・QR用のトークン |
| チェックイン / コネクション | チェックインは「現場にいる」状態。コネクションは「つながった」という記録 |
| コネクション（一方向） / フォロー（相互） | なふだのコネクションは一方向。双方向のつながりは2行のレコードで表す |
