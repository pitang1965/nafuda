# なふだ — ドメイン用語集

このファイルはドメイン語彙の唯一の定義源。実装の詳細や仕様は書かない。

---

## 用語定義

### なふだ
ユーザーが他者に見せるプロフィールカード。アプリ名でもある。コード上は **Persona** と呼ぶ。UI上では常に「なふだ」と表記する（「ペルソナ」はユーザーに見せない）。1ユーザーが複数持てる（例：「婚活用なふだ」「車趣味用なふだ」）。「マイなふだ」は自分のなふだ一覧を指す。

### ペルソナ (Persona)
「なふだ」のコード上の名称。`personas` テーブルに格納。1ユーザーにつき1つが `isDefault = true` のデフォルトペルソナとなり、チェックインの主体になる。コネクションは「つながる」時にユーザーが選んだペルソナが主体になる（複数ペルソナがある場合はピッカーを表示、1つのみの場合は自動選択）。UI上では「なふだ」と呼ぶ。

### ラベル (Label)
なふだの用途を自分だけが識別するための名前。**オーナーにのみ表示され、公開プロフィールには出ない**。「推し活用」「婚活用」「仕事用」など自由記述。コード上は `label`。`personas.label` に格納。任意項目（未設定の場合は displayName のみ表示）。ウィザードで作成する初回なふだにはデフォルト値「メイン」が入力欄にセットされる。なふだ作成時に入力欄を用意し、未設定のなふだには設定を促すリンクを表示する。

### UrlId
ユーザーに1つ割り当てられる英数字識別子。**システムが生成するランダム文字列**であり、ユーザーは選択できない。変更不可。`url_ids` テーブルに格納。`/u/{urlId}` 単独のルートは存在しない。すべてのなふだへのアクセスは ShareToken を伴う URL 経由のみ。

### ShareToken
ペルソナごとに発行されるランダムな16進数トークン。QRコードのURLに埋め込まれ、プロフィール参照に使われる。デフォルトペルソナも含め、すべてのペルソナが固有の ShareToken を持つ。`personas.share_token` に格納。

### QRコード
`/u/{urlId}/p/{shareToken}` 形式のURLをエンコードした画像。デフォルトなふだ・用途別なふだを問わず、すべてのなふだがこの形式を使う。これを他者が読み取ることでプロフィールを閲覧できる。スキャン自体ではコネクションは記録されない。

### イベント (Event)
誰かが企画・作成し、参加者がチェックインできる時間と場所のある催し。ライブ・コンサート・ファンミ・オフ会・勉強会など。スラッグ（URL-safe文字列）で識別される。主催者（または参加者）が `/events/new` から明示的に作成する。

**イベントに含まれないもの：**
- 個人の聖地巡礼・撮影地訪問（個人の行動であり、主催者のいる催しではない）
- 趣向・コミュニティ（「○○ファン」「車中泊グループ」等）

聖地を使った企画催し（「○月○日に聖地○○に集まれ！」等）は主催者が作成したイベントとして扱う。個人の聖地訪問を記録したい場合は推しタグで代替する。

`events` テーブルに格納。

### チェックイン (Checkin)
ペルソナがイベントに「入る」アクション。`checkedOutAt IS NULL` の状態がアクティブ。新しいチェックイン時に直前のチェックアウトが自動記録される（同時に複数イベントへの参加不可）。`event_checkins` テーブルに格納。

### コネクション (Connection)
「つながる」ボタンを押した側（fromPersona）から、QRを見せた側（toPersona）への一方向の記録。双方向のつながりは2行で表現される。「つながる」をタップすると、複数のなふだを持つユーザーにはペルソナピッカーを表示し、どのなふだとして接続するかを選ぶ（1つのみの場合は自動選択）。各コネクションには「出会いの文脈」（いつ・どこで）が付与される。文脈はチェックイン中の自動付与、GPS自動記録、または後からの手動編集（場所名・イベント名のフリーテキスト）で設定できる。正式なイベントエンティティへの後付け紐付けはしない。コネクション一覧の「どのなふだとして」表示には、接続時点のラベルではなく**現在のラベル**を使う（ペルソナテーブルをJOINして取得）。ラベルを変更すると過去のコネクション表示も更新される。`connections` テーブルに格納。

### 同担拒否 (Dojin Reject)
同じ推しを応援するファン（同担）との交流を望まないというユーザーの意思表示。`personas.dojin_reject = true` の場合、そのペルソナはイベント参加者一覧から非表示になる。

### 推しタグ (Oshi Tags)
ペルソナに紐づく自由記述のタグ配列。応援しているアーティスト・キャラクター・作品などを登録する。任意項目（未設定のまま使用できる）。アプリは推し活を主なユースケースとして打ち出しているが、趣味・コミュニティ・仕事など汎用的な名刺交換用途も想定している。`personas.oshi_tags` に格納。

### SNSリンク (SNS Link)
ペルソナに紐づく外部サービスへのリンク。プラットフォーム種別・URL・表示順を持つ。`sns_links` テーブルに格納。対応プラットフォーム: x, instagram, tiktok, youtube, discord, line_openchat, github, spotify, facebook, minkara, linkedin, note, pixiv, other。

### 公開範囲 (Field Visibility)
ペルソナのフィールドごとの公開/非公開設定。`personas.field_visibility` に JSON で格納。デフォルトは全フィールド公開（QRを渡す行為自体が公開の意思表示という前提）。

## 重要な区別

| 混同しやすいペア | 区別 |
|---|---|
| なふだ / ペルソナ | 同じ概念。UIでは「なふだ」、コードでは「Persona」と呼ぶ。ユーザーに「ペルソナ」という語は見せない |
| UrlId / ShareToken | UrlIdはユーザー単位・変更不可の識別子（単独でアクセス不可）。ShareTokenはペルソナ単位・QR用のトークン。プロフィールへのアクセスには必ず両方が必要 |
| チェックイン / コネクション | チェックインは「現場にいる」状態。コネクションは「つながった」という記録 |
| コネクション（一方向） / フォロー（相互） | なふだのコネクションは一方向。双方向のつながりは2行のレコードで表す |
| displayName / ラベル | displayNameは他者に見せる公開名。ラベルはオーナーだけが見る自己識別用の名前（最大20文字）。コネクション一覧など自分向けの画面ではラベルを優先表示し、未設定の場合は displayName にフォールバック |
