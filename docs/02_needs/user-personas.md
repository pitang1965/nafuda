# ユーザーペルソナ (User Personas)

> ペルソナは「平均的なユーザー像」ではなく「解くべき課題を持つ典型的なユーザー」を描く。

---

## ペルソナ A（メイン）：もちこ（23歳・推し活ユーザー）

| 項目 | 内容 |
|------|------|
| 名前（仮名） | もちこ（ハンドルネーム：もち@担当箱推し） |
| 年齢 / 性別 | 23歳 / 女性 |
| 職種 / 役割 | 会社員（事務職） |
| 所属 / 環境 | 関東在住。週末を中心にライブ・舞台・ファンミに参加。遠征も年に数回。 |

### 日常のワークフロー（現場参加時）

1. **参加前**：セトリ予習・現場装備（うちわ・ペンライト）を準備する。
2. **当日**：隣の席や待機列で会話が弾み「また現場で会いたい」と思う人に出会う。
3. **SNS交換の場面**：「X のアカウント教えてください」と口頭で伝えるが、会場の騒音で聞き取れなかったり、打ち間違えたりする。また、本名が出てしまう X アカウントを教えることへの抵抗感もある。
4. **参加後**：誰と出会ったか記憶が薄れる。後日フォロバされても「誰だっけ」となる。

### ペインポイント

- **口頭 SNS 交換の煩雑さ**：騒がしい会場で ID を伝えるのは聞き取りミスが多く、気まずい。
- **プライバシーへの不安**：本名が紐づく SNS アカウントを初対面に教えることへの抵抗がある。
- **出会いの文脈が消える**：「あの日、同じ現場にいた」という共通体験が後から確認できない。
- **同担拒否への配慮**：相手が同担歓迎か拒否かを確認せず交換してトラブルになるリスクがある。

### ゴール

- **スムーズなつながり形成**：QR 一発でハンドルネームと推し情報をその場で共有したい。
- **文脈の保存**：「○○ライブで出会った人」として後から見返せるようにしたい。
- **プライバシーの保護**：本名・本アカウントを出さずに推し活仲間とつながりたい。
- **同担設定の明示**：自分が同担歓迎 / 拒否であることを事前に相手に伝えたい。

## ペルソナ B（メイン）：れい（28歳・聖地巡礼ユーザー）

| 項目 | 内容 |
|------|------|
| 名前（仮名） | れい（ハンドルネーム：れい⛩撮影地巡り） |
| 年齢 / 性別 | 28歳 / 性別非公開 |
| 職種 / 役割 | フリーランス（イラストレーター） |
| 所属 / 環境 | 地方在住。アニメ・ゲームの聖地を写真撮影しながら巡礼。SNS での発信が趣味。 |

### 日常のワークフロー（聖地巡礼時）

1. 撮影スポットで同じ作品のファンに出会い、作品トークで盛り上がる。
2. 「どのシーンを撮りに来たんですか？」から会話が始まり、関係を続けたいと思う。
3. 相手が Instagram や Pixiv のアカウントを持っているか不明。SNS の種類もバラバラで共通の交換手段がない。
4. 別れた後、また同じ場所に来たときに「あの人がいたらいいのに」と思う。

### ペインポイント

- **SNS プラットフォームの分散**：相手の主戦場（X / Instagram / Discord / Pixiv）がわからない。
- **つながりのきっかけの薄さ**：共通点（作品・聖地）は明確なのに、それを一目で伝える手段がない。
- **再会のしにくさ**：「同じ聖地に行くとき教え合う」関係を作りたいが、維持する仕組みがない。

### ゴール

- **SNS リンクを一括共有**：X・Instagram・Discord などを相手に一度に見せたい。
- **訪問した聖地を記録**：「○○神社（△△ロケ地）に行きました」という文脈をプロフィールに残したい。
- **共通点を可視化**：「同じ作品のファン」であることをその場で一目でわかるようにしたい。

## ペルソナ C（サブ）：ヤマダ ヒロシ（50代・Web ディレクター）

> ※ 推し活ユーザーを先行で磨いた後、一般イベント向け展開の際にメインターゲットになる想定。

| 項目 | 内容 |
|------|------|
| 名前（仮名） | ヤマダ ヒロシ |
| 年齢 / 性別 | 50代 / 男性 |
| 職種 / 役割 | フリーランス Web ディレクター（副業から本格化） |
| 所属 / 環境 | 個人事業。受注・設計・ディレクション中心。デザイン・コーディング・マーケティング視点も持つ。 |

### 経歴

50歳を過ぎてから Web 制作を開始。時給300円の初案件からスタートし、継続受注を重ねて現在は単価40万円クラスの案件まで成長。現在も定期的に Web 案件を受けながら制作を続けている。

### 日常のワークフロー（イベント参加時）

1. **参加前**：Connpass 等で勉強会・オフ会を見つけて登録する。
2. **当日**：名刺や口頭で自己紹介するが、参加者が多いと全員とは話せずに終わる。
3. **参加後**：誰と話したか、どんな人だったかを思い出せない。連絡先を交換しても記憶が薄れていく。

### ペインポイント

- **つながりの薄さ**：多くのイベントに参加しても実際につながれる人数が少なく、関係が深まりにくい。
- **記憶の限界**：顔・名前・会社・得意技術・趣味を一度に覚えきれない。
- **自己開示の難しさ**：自分の経歴やスキルを相手に効果的に伝える手段が乏しい。

### ゴール

- **広いつながり**：1回のイベントで多くの人と情報交換できるようにしたい。
- **情報の記録・参照**：出会った人の情報をその場で記録し、後から見返せるようにしたい。
- **印象の定着**：自分の経歴・スキルを相手に分かりやすく伝え、覚えてもらいたい。

| 項目 | 内容 |
|------|------|

| 年齢 / 性別 | 37歳 / 男性 |
| 職種 / 役割 | フリーランス Web エンジニア、個人アプリ開発者 |
| 所属 / 環境 | 大阪・天満エリア在住。「街のいい店マップ」アプリを個人開発中。 |

### 経歴

20代前半から大型トラック運転手として全国を走り回る。30歳頃に物流倉庫マネージャーに昇進し、現場効率化を担う中でプログラミングに興味を持つ。34歳頃に独学で Web エンジニアへ転身。現在はフリーランス寄りでアプリ開発を続けながら、飲み屋でのコミュニケーション力を磨いている。

### 日常のワークフロー（イベント参加時）

1. **企画**：天満・中崎町エリアの焼肉・夜カフェ・バーなどのイベントを自ら企画・主催する。
2. **参加**：散歩会や飲み会に積極的に参加し、場を盛り上げる。
3. **つながり**：話のきっかけを作りたいが、初対面の人と自然に会話を続けるのが難しい。

### ペインポイント

- **話のきっかけ不足**：初対面でどう切り出すか迷い、会話が弾まないことがある。
- **自己紹介のぎこちなさ**：経歴がユニーク（元トラック運転手→エンジニア）で面白いのに、うまく伝えられない。
- **関係の継続**：飲み会で出会った人との関係をその後に維持するのが難しい。

### ゴール

- **話のきっかけ**：アプリでプロフィールを見せることで、自然な会話のとっかかりを作りたい。
- **個性の伝達**：ユニークな経歴や趣味を視覚的・分かりやすく相手に伝えたい。
- **緩やかなつながり**：イベント後も気軽に連絡が取れる関係を築きたい。

### パーソナリティ

- 関西弁（マイルドな大阪弁）で明るく親しみやすいトーン。
- 16Personalities：「運動家タイプ（ENFP）」— 行動派で飲み会・イベントを盛り上げるのが得意。
- YouTube に不定期で「元運転手が教える飲み屋効率化術」や大阪の隠れ家スポット紹介を投稿。
