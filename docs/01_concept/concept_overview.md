# なふだ — コンセプト概要

> 名称: **なふだ**
> 命名由来: 江戸時代の「名札」文化から。歴史的にも「身元を示し、場でつながる札」の意を持つ和語。

---

## 一言コンセプト

**「出会いに、文脈を。」**

イベント・聖地・推しでつながる、**文脈型デジタル名刺アプリ**。
いつ・どこで・誰を応援しに来て出会ったのか——その "文脈" をまるごと残せる現代版の名札。

---

## サービス概要

ライブ会場・聖地・オフ会で隣り合った人と、口頭でSNS IDを交換する手間なく、ワンタッチでつながれるデジタル名刺アプリ。匿名・ハンドルネーム前提の設計で、リアル個人情報を晒さずにファン同士のつながりを作れる。

「推し」「ジャンル」「参加中のイベント」といった**出会いの文脈**をプロフィールに残せるため、後から見返したときに「あの日、同じ現場にいた人」として記憶に残る。

推し活を主軸にしつつ、勉強会・交流会など一般イベントにも同じ仕組みで利用できる汎用性を持つ。

### 主要機能

| 機能 | 概要 |
|------|------|
| 推し・ジャンルタグ登録 | プロフィールに「推し」と「ジャンル」を複数タグで登録 |
| イベント／聖地チェックイン | ライブ・現場・聖地巡礼先でチェックインし、出会いの文脈を残す |
| QR ワンタッチ交換 | 口頭でのSNS ID交換を置き換える1タップ共有 |
| SNSリンク一括共有 | X / Instagram / Discord / LINEオープンチャット 等をまとめて提示 |
| 同じ現場にいた人一覧 | 「あの時いましたよね？」の共感体験を可視化 |
| 複数プロフィール | 推し活用・本業用・勉強会用などシーンで使い分け |
| 公開範囲・同担設定 | 同担歓迎／同担拒否などの意思表示、プロフィール単位の公開制御 |
| **イベント一括つながり** | ★差別化点：現場に入ったらその場の全員と一括でつながれる |

### 差別化ポイント

一般的な名刺アプリが「仕事の肩書き交換」に最適化されているのに対し、**なふだ**は「**いつ・どこで・誰の現場で**出会ったか」という文脈を第一級の情報として扱う。

* 「あの日、同じ聖地にいた」という共感が残る
* 推し・ジャンルタグで会話のきっかけが生まれる
* 匿名・ハンドルネーム前提でプライバシーに配慮
* 口頭で SNS ID を伝える気まずさ・ミスを排除

## ターゲット／ユースケース

### メイン：推し活ユーザー

* **ライブ・コンサート会場**：隣の席・参戦組との「また会いたい」を残す
* **聖地巡礼・撮影地巡り**：同じ日に訪れた人との共感接続
* **ファンミ・特典会・オフ会**：初対面の同担と気まずさなくSNS交換
* **遠征**：遠方から来たファン同士の交流拠点づくり

ジャンルはアイドル・アニメに限らず、VTuber・俳優・声優・2.5次元・スポーツ・Kポップ・バンド・スポーツ選手など広く想定。

### サブ：一般イベント

同じ仕組みで以下のような一般イベントにも使える：
* 勉強会・カンファレンス・LT会
* 異業種交流会・地域イベント
* 飲み会・オフ会・散歩会

※ 推し活ユースを先行で磨き込み、そこで鍛えた「文脈型名刺」の価値を汎用イベントに横展開する方針。

## ユーザー設計上の原則

推し活領域の文化特性を反映し、以下を必須要件とする：

| 原則 | 内容 |
|------|------|
| 匿名・ハンドル前提 | 本名・所属の入力は任意。ハンドルネーム／アバターでの運用を基本とする |
| オプトイン制 | 「同じ現場にいた人」等のマッチングは、見つけられたい人だけが表示される設定 |
| 最小登録での利用 | アカウント登録なしでもプロフィール閲覧・QR読み取りまで可能な軽さを目指す |
| 同担拒否への配慮 | 同担拒否の意思を設定で明示できる／一覧表示から外せる |
| プロフィール分離 | 推し活用と本業用を完全に別プロフィールとして運用できる |
| 個人情報の粒度選択 | 名前・会社・連絡先などフィールド単位で公開／非公開を制御 |

## 機能要件

推し活視点の要点まとめから、機能要件として以下を明確化：

1. **推し・ジャンルのタグ登録**（自由記述＋サジェスト）
2. **イベント・聖地ごとのチェックイン機能**（日付＋場所＋イベント名）
3. **QRコードでのワンタッチ名刺交換**
4. **SNSリンクの一括共有**（X / Instagram / Discord / LINE オープンチャット / TikTok 等）
5. **「同じイベントにいた人」の一覧表示**
6. **同担設定・公開範囲設定**
7. **複数プロフィール切り替え**
8. **名札ケース装飾**（Phase 2以降）：他者が閲覧するプロフィール画面に、レース・リボン・ラインストーン等の「痛名札ホルダー」風フレーム装飾を適用できる。推しカラーと連動したテーマ選択も想定

## ロードマップ（想定）

### Phase 0 — プロトタイプ（UX検証）
* DB不使用・フロントエンドのみ
* イベントルーム画面（参加者カードがアニメーションで出現）
* PWA 1タップインストール
* モックログイン（タップでログイン状態をシミュレート）
* QRコードはモバイルのネイティブカメラで読み取り → URL遷移（アプリ内スキャナーなし）
* 推し・ジャンルタグ表示（モックデータ）

### Phase 1 — 初期リリース（推し活フォーカス）
* プロフィール作成・QR共有
* 推し・ジャンルタグ登録
* イベント／聖地チェックイン
* 「同じ現場にいた人」一覧
* 同担設定・公開範囲設定
* ソーシャルログイン実装（匿名継続利用にも配慮）

### Phase 2 — 拡張
* ユーザー間のメッセージ交換
* 一般イベント（勉強会・交流会）向けの導線強化
* アプリ内QRスキャン

## 設計方針

* **文脈ファースト**：「いつ・どこで・誰の現場で出会ったか」を一級の情報として扱う
* **匿名ユーザーファースト**：ハンドルネーム／アバター運用を前提とした UI・言葉選び
* **オプトインUX**：探される・見つかるは常に自分の意思で切り替えられる
* **見せたくなるデザイン**：推しカラー・ジャンル別テーマ・名札ケース風フレーム装飾など、プロフィール画面にアイデンティティを込められる
* **推し活から汎用へ**：まず推し活で刺さる体験を作り、同じ仕組みを一般イベントへ展開

## 名称「なふだ」について

| 観点 | 内容 |
|------|------|
| 由来 | 江戸時代の「名札（なふだ）」文化。身元を示し、場でつながる札 |
| コンセプト整合 | 「文脈をつなぐ札」という本アプリのコアと歴史的背景が合致 |
| 表記 | ひらがな「なふだ」を基本。デザイン的に「名札」表記の併用も検討 |
| 商標 | 取得可能性が高い状況（要確認：J-PlatPat 等での正式調査） |
| ドメイン | 取得可能性が高い状況（要確認：各 TLD の空き状況） |

> ※ 正式採用にあたっては商標・ドメインの最終確認を別途実施する。
