# なふだ SEO・打ち出しプラン

> 戦略の決定経緯は [ADR-0009](./adr/0009-event-first-positioning.md)、用語定義は [CONTEXT.md](../CONTEXT.md) の「打ち出し」を参照。
> このファイルは実装のための working material（コピー案・キーワード設計）であり、用語集ではない。

## 戦略サマリー

- **問題**: アプリ名「なふだ」は一般名詞（名札）で Google 指名検索が埋もれる。ドメイン `nafuda.me` は変更不可。
- **方針**: SEO は **課題検索（A-2）を主軸（70%）／指名検索の防衛（A-1）を最低限（30%）**。
- **楔**: 「名刺を持たない／出さない場での出会い」＝ **“仕事じゃない”出会い**。ビジネス交流会は対象外（Eight の縄張り）。
- **差別化の一文**: **Eight＝仕事の名刺。なふだ＝“仕事じゃない”出会いの名刺。**
- **楔は狭く・射程は汎用のまま**（Discord 型）。推し活は表の顔ではなく「最も濃い実例」として配置。

## 1. ヒーローコピー案

| 案 | キャッチ | サブ | トーン |
|---|---|---|---|
| **A（採用）** | **その場で会った“仕事じゃない”人と、SNSでつながる。** | 勉強会、オフ会、飲みの席。紙の名刺は出さないけど、また会いたい人がいる。QRを見せるだけ。 | 共感・具体 |
| B | **名刺を出さない場の、名刺。** | 電話番号でもメールでもなく、SNSで。本名を出さなくてもいい。 | 端的・コンセプト |
| C | **「またどこかで」を、つながりに。** | イベントで隣になった人と、QRひとつでSNS交換。 | 情緒 |

採用はA。「仕事じゃない」「紙の名刺は出さない」で Eight との境界線を即座に引き、対象4イベントを具体名で内包できるため。

## 2. SEOキーワード表（A-2 課題検索の核）

1ページ＝1クラスタ。各イベント種別ごとに独立したランディング/記事ページを作る。

| ページ | 主クエリ | ロングテール | 殺し文句 |
|---|---|---|---|
| **勉強会・技術系** | `勉強会 連絡先 交換` | `もくもく会 つながる` / `カンファレンス 名刺持ってない` / `LT会 SNS 交換` | 名刺がなくても GitHub/X を一発で渡せる |
| **趣味・オフ会** | `オフ会 連絡先 交換` | `オフ会 本名 出さない` / `趣味 仲間 SNS つながる` / `初対面 LINE抵抗` | 本名・電話番号を出さず SNS だけ渡せる／用途別なふだ |
| **懇親会・パーティー** | `懇親会 つながる qr` | `立食パーティー 連絡先` / `交流イベント また会いたい` | QRを見せるだけ・相手のアプリ不要で閲覧可 |
| **バー・居酒屋** | `バー 出会い sns 交換` | `飲み屋 連絡先 渡し方` / `居酒屋 隣 仲良く` | その場限りにしない・気軽・SNS中心 |
| **推し活（最濃の実例）** | `推し活 名刺` | `同担 つながる` / `ライブ 連絡先 交換` / `推し友 作り方` | 同担拒否・推しタグ。競合空白で1位を取りやすい |

> ビジネス交流会クラスタ（`デジタル名刺` `名刺アプリ` 等）は **意図的に狙わない**。Eight・Sansan のレッドオーシャンで勝てないため。

## 3. A-1 指名検索の防衛（低コスト併設）

- 全ページの `<title>` を `〇〇｜なふだ - "仕事じゃない"出会いのデジタル名刺` 形式に固定。
- `なふだ アプリ` / `なふだ 名刺` を meta description・OGP・構造化データ（`SoftwareApplication` JSON-LD）で押さえ、「名札グッズ」と棲み分ける。
