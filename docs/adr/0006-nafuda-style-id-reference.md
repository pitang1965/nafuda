# ADR-0006: なふだスタイルはIDのみDBに保存し、定義はコード管理とする

- **ステータス**: 採用
- **決定日**: 2026-06-02

## 背景

なふだ（Persona）に視覚的な装飾（背景・フォント・SVGフレーム）を適用する「なふだスタイル」機能を追加するにあたり、スタイル情報の保存方法を決める必要があった。

## 問題

スタイル情報の保存には大きく2つの方針がある。

1. **スタイルIDのみ保存**: `personas.style_id = 'sakura'` のように文字列IDだけをDBに持ち、実際のCSS定義・SVGアセットはコードで管理する
2. **スタイルプロパティをJSONで保存**: `personas.style = { background: '...', fontFamily: '...', frameId: '...' }` として各要素の値をDBに直接保存する

## 決定

**スタイルIDのみDBに保存する（方針1）**。

`personas.style_id TEXT NULL` カラムを追加し、`null` は「スタイル未選択（デフォルト白背景）」を意味する。IDの実体（CSS変数・SVGアセットパス・フォント名・`isFree`フラグ）はコード側の定数オブジェクトとして管理する。

## 却下した代替案

- **JSONプロパティ保存**: スタイルの見た目を後から修正しても既存ユーザーに反映されず、「sakuraの背景色を変えたい」ときに全ユーザーのデータを更新するか、古い値のまま動かすかという問題が生じる。また有料チェックのロジックがDB値とコード定義に分散する。

## トレードオフ

- **失うもの**: ペルソナごとに細かいプロパティを調整する自由度（個別カスタマイズ）
- **得るもの**: スタイル定義の一元管理・全ユーザーへの一括反映・有料フラグ等のロジックをコードに集約できるシンプルさ
- **許容できる理由**: 当面はパッケージ選択方式（個別カスタマイズなし）で進めることが確定しており、柔軟性よりも管理のシンプルさを優先する
