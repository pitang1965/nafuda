# ADR-0012: 即時イベント文脈は双方のコネクションにコピーし、各自が編集できる

- **ステータス**: 採用
- **決定日**: 2026-06-14

## 背景

「なふだを交換する」フロー中に作成される即時イベント（「今どこで？」で入力する出会いの文脈）は、当初 **A（QR を発行した側）のコネクション行にのみ** 文脈を付与し、B（スキャンした側）の行には付与しない設計だった（旧 CONTEXT.md ②）。理由は「B はスキャン時に A が書いたメモを見ていないため、B は自分の言葉で後付け（③）すべき」という認識論的な配慮だった。

しかしこれは、つながり一覧で経路ではなく当事者によって表示が変わる非対称を生む。同じ1回の交換なのに、A の画面にはイベント名が出るが B の画面は「誰として・いつ」だけになる。B 側は空欄スタートで、③の導線も弱く、実際には誰も書かない。さらに即時イベント由来の文脈は `eventId` 付きのため編集ロックされ、作成者 A 自身ですら後から書き換えられなかった。

加えて、コネクションは ADR-0007 で **A→B と B→A の独立した2行**として保持しており、削除も各自が自分の行を独立して行う（非対称削除）。文脈もこの「各行は持ち主のもの」という思想に揃えるのが自然である。

## 決定

即時イベントの文脈（イベント名・場所名・日時）を、**A と B 双方のコネクション行に `eventId` 付きでコピーする**。即時・企画とも双方に付与する点は共通で、**差は編集ロックの有無に集約する**。

- **編集ロックの判定を `eventId !== null && isInstant === false` に変更する**。企画イベント由来のみ固定（公開ページを持つ客観的事実）、即時イベント由来は各自が③で自分の行を編集できる。
- 即時イベントの `eventId` は**破棄せず保持する**。将来「同じ即時イベントに関連する複数人」を `eventId` で引けるよう、出会いの正本イベントへのリンクを残す。
- B の初期値は A が「今どこで？」に書いた文脈。ただしコピーは **成立時点のスナップショット**であり、以後 A・B 各自の編集は相手の行に伝播しない（各行は独立した記録）。編集はコネクション側の非正規化コピー（`event_name` / `venue_name`）にのみ書き込み、`events` 本体には影響しない。
- 編集後はコネクション側の名前が `events.name`（正本）と乖離しうるが、これは意図的な役割分担である（`eventId`＝出会いの正本イベント／コネクション側の名前＝各自の私的ラベル）。

ロック判定に `isInstant` が必要になるため、`getMyConnections` と `updateConnection` は `events` を LEFT JOIN して `isInstant` を取得する。スキーマ変更・データマイグレーションは不要——既存の即時イベント由来コネクションは `eventId` 付きのまま、ロック判定の変更だけで編集可能になる。

## 却下した代替案

- **現状維持（B には付与しない）**: 指摘どおりの非対称が残る。B の画面が空欄のままになる痛点を解消できない。A 自身も編集できないままになる。
- **対称ロック（企画と同じ扱い）**: 双方に `eventId` 付きでロックコピーする。実装は単純だが、即時イベント名は公開ページを持たない A のその場の私的ラベルであり、それを客観的事実として固定するのは強すぎる。各自が自分の記憶として直せない。
- **`eventId` を null 化してフリーテキスト化**: 即時イベント由来コネクションの `eventId` を null にすれば、既存のロック判定（`eventId !== null`）を変えずに編集可能になる。実装は最小だが、**将来の複数人関連に使える即時イベント識別子を能動的に破棄してしまう**ため却下。リンク保持のコストは JOIN 1 本に過ぎない。
- **共有元 + 伝播（ライブ JOIN / 編集時再同期）**: 即時イベントを A 所有のミニ企画イベントとして扱い、A の編集を B へ伝播させる。新たな編集 UI・伝播機構が必要で、各自が独立して持つ ADR-0007 の思想とも合わない。

## トレードオフ

- **失うもの**: 即時イベント文脈の「事実としての固定」。A・B のコピーが編集により乖離し、さらにコネクション側の名前が `events.name` 正本とも乖離しうる。
- **得るもの**: つながり一覧の当事者間非対称の解消。A 自身による文脈の訂正。ADR-0007 の「各行は持ち主が独立して持つ」思想との一貫性。即時イベント識別子の保持による将来拡張（複数人関連）の余地。データマイグレーション不要。
- **許容できる理由**: 即時イベント文脈は公開ページを持たない私的な覚え書きであり、A・B それぞれが自分の記憶として保持・編集できることに価値がある。両者にしか見えず、不快なら削除もできる（ADR-0007 の非対称削除）。乖離は「`eventId`＝正本／コネクション名＝私的ラベル」という役割分担として許容される。
