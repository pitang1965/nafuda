# ADR-0009: 表の顔を「推し活ファースト」から「“仕事じゃない”イベント出会いファースト」へ転換する

- **ステータス**: 採用
- **決定日**: 2026-06-13

## 背景

「なふだ」というアプリ名は一般名詞（名札）であり、Google での指名検索（`なふだ`）が名札グッズに埋もれて発見されづらい。ドメイン `nafuda.me` は変更できない。この発見性をどう改善するかを検討する中で、SEO 戦略とプロダクトの「表の顔（ヒーロー・第一印象・打ち出し）」を見直すことになった。

検討の結果、以下が確認された。

- SEO は **課題検索（カテゴリ/インテント検索）を主軸**に取りに行く。一般名詞「なふだ」と正面から戦うより迂回する方が筋が良い。
- 「デジタル名刺」「QR名刺アプリ」といった**汎用名刺クエリは Eight・Sansan の壁**で個人開発が勝てる余地がない。
- これまで CONTEXT.md / README は **推し活を主なユースケースとして打ち出していた**が、**作者本人がこのアプリを実際に使いたい動機は推し活ではなくイベント全般**だった。打ち出しと作者の本音（＝dogfooding できる用途）が乖離していた。

## 問題

SEO の楔とプロダクトの「表の顔」を、どのユースケースに合わせるか。

1. **推し活ファースト（現状維持）**: 熱量・拡散力は最強だが、作者の本音と乖離し、汎用ユーザーが「推し活アプリ」と誤解して離脱する
2. **イベント出会いファースト（転換）**: 作者が dogfooding でき、汎用性が最初から伝わるが、推し活の拡散の爆発力を表で使えなくなる

なお「楔（SEOの入口）は狭くする」「プロダクトの射程は汎用のまま保つ」という2点は前提として確定しており、両者は両立する（Discord がゲーマーという狭い楔から汎用に広げた構造と同じ）。本ADRが決めるのは**表の顔をどの楔に合わせるか**である。

## 決定

**表の顔を「その場で会った“仕事じゃない”人と、SNSでつながる」に作り替える（方針2）。**

- **背骨（ヒーロー）**: 「名刺を持たない／出さない場での出会い」を核に据える。仕事じゃないから紙名刺を持っていない、電話番号やメールではなく SNS でつながりたい、本名や勤務先は出したくないこともある —— この文脈を表現する。
- **対象イベント**: 勉強会・技術系 / 趣味・オフ会系 / 懇親会・パーティー系 / バー・居酒屋での出会い。**ビジネス交流会は対象から外す**（紙名刺＋Eight の縄張りであり、なふだの戦場ではない）。
- **差別化の一文**: 「Eight＝仕事の名刺。なふだ＝“仕事じゃない”出会いの名刺。」
- **推し活の扱い**: 表の顔ではなく、**最も濃い実例コンテンツ**として一等地に残す。SEO 上は `推し活 名刺` を最濃のロングテール記事として配置し、拡散力を素材として回収する。
- **SEO の主軸**: 具体的なイベント種別 × 連絡先交換のロングテール（`勉強会 連絡先 交換`、`オフ会 連絡先 交換 本名出さない`、`バー 出会い SNS 交換`、`懇親会 つながる QR` など）。抽象的な「イベント」単体（Peatix/connpass と衝突）は狙わない。
- **汎用性の保持**: 趣味・コミュニティ用途などの汎用性は**プロダクトの射程と二次メッセージに残す**。SEO の核には据えない。

## 却下した代替案

- **推し活ファースト維持**: 熱量・拡散力・競合空白という強みはあるが、(1) 作者が dogfooding しづらく改善のフィードバックループが回らない、(2) 「推し活アプリ」という誤解で汎用ユーザーが離脱しファネルに穴が空く、(3) 作者の本音（イベント全般）との乖離が将来の意思決定をぶれさせる。強みはコンテンツ層（最濃の実例）で回収できるため、表の顔である必要はない。
- **汎用名刺ファースト**: 「デジタル名刺」市場は Eight・Sansan のレッドオーシャンで個人開発が勝てない。SEO の選択肢としてそもそも成立しない。
- **抽象的な「イベント」ファースト**: 「イベント」単体は Peatix・connpass と検索意図が衝突し、メッセージも抽象的で刺さらない。具体的なイベント種別に落とす必要がある。

## トレードオフ

- **失うもの**: 推し活というラベルの拡散の爆発力・SNSでの言及されやすさを、表の顔として直接使えなくなる。メッセージがやや抽象的になりやすい（具体イベント名で殴ることで緩和する）。
- **得るもの**: 作者本人が dogfooding できる（改善ループが回る）。汎用性が最初から伝わり流入後の離脱が減る。SEO の楔・プロダクトの本音・打ち出しが一直線に揃う。Eight との差別化が一文で言えるようになる。
- **許容できる理由**: 作者が日常的に使い込めないプロダクトは育たない。推し活の強みはコンテンツ（最濃の実例）として回収可能であり、表の顔を明け渡しても失われない。SEO上も汎用名刺で勝てない以上、勝てるニッチ（“仕事じゃない”出会い）に資源を集中するのが合理的。
