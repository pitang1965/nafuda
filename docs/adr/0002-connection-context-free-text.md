# ADR-0002: コネクション文脈の後付け編集はフリーテキストにする

- **ステータス**: 採用
- **決定日**: 2026-05-27

## 背景

チェックインなしで「つながる」を実行するケース（バーでの名刺交換など）では、コネクションに付与される文脈が日時のみになる。ユーザーが後から「渋谷・バー○○ ❎️❎️オフ会」のように文脈を補いたいという需要がある。

## 決定

コネクションの場所名・イベント名フィールドをフリーテキストで後付け編集できる機能を追加する。編集内容は既存の `connections.venue_name` / `connections.event_name` フィールドに書き込む。

また、チェックインなしで「つながる」を実行した際は、ブラウザの位置情報許可が得られれば GPS 座標を暫定文脈として自動記録する（`connections.gps_coordinates` を追加）。

## 却下した代替案

**後付けでイベントエンティティに紐付ける**: バーでの偶発的な出会いに公式イベントを作るのは過剰。「❎️❎️オフ会」が本当に組織的なイベントであれば主催者が事前にイベントを作ってチェックインするフローが自然であり、コネクション編集とは別の経路で対応できる。

## トレードオフ

- **失うもの**: 「同じバー・オフ会にいた他の人を発見する」機能（Event エンティティへの紐付けがないため）
- **得るもの**: シンプルな実装、ユーザーが覚えているうちに気軽にメモできる UX
- **許容できる理由**: 偶発的な出会いの文脈は「後から見返したときに思い出せる」程度で十分。「同じ場所にいた人の発見」はチェックイン流入のユースケースが担う
