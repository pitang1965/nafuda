# ADR-0004: コネクション一覧のラベル表示にスナップショットではなくライブJOINを使う

- **ステータス**: 採用
- **決定日**: 2026-05-28

## 背景

コネクション一覧の「〇〇 として」欄には、接続したなふだ（fromPersona）のラベルを表示する。ラベルはペルソナに後から変更できるため、「接続時点のラベルを保存するか」「現在のラベルをJOINで取得するか」の選択が生じた。

## 決定

`getMyConnections` では `fromPersona.label` をライブJOINで取得する。`connections` テーブルにラベルのスナップショット列は持たない。

## 却下した代替案

- **スナップショット保存**: `connections.from_label` 列を追加し、接続時点のラベルを非正規化して保存する。ラベルを変更しても過去の表示は変わらない。スキーマが増え、ラベル変更時の整合性管理が不要になるが、「今自分がそのなふだをどう呼んでいるか」と「過去の記録」が乖離するケースが生まれる。

## トレードオフ

- **失うもの**: 接続時点のラベルの不変性。ラベルを「メイン→推し活用」と変えると過去のコネクション表示も変わる
- **得るもの**: スキーマのシンプルさ。「今の自分のなふだ認識」が一覧に常に反映される
- **許容できる理由**: ラベルはオーナーだけが見るプライベートな自己識別名。過去の接続文脈（いつ・どこで会ったか）はイベント名・場所名で保持されており、「どのなふだとして」の履歴的正確性よりも「今の自分の言葉で見る」体験を優先する
